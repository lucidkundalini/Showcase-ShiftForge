# Public Scope

## Included

- selected backend modules
- verified live evidence
- high-level architecture
- truthful implemented-vs-intended boundary

## Excluded

- `.env` files
- provider keys
- encrypted or raw user session data
- full private deployment tree
- private internal hardening/sync modules that are not ready for public release
- code that would expose unnecessary operational attack surface

## Why This Repo Is Curated

The private ShiftForge tree is not appropriate to publish as-is.

A truthful public repo should:

- show real behavior
- avoid false claims
- avoid leaking secrets
- avoid promoting dormant code as production code
- stay readable for investors and technical reviewers
